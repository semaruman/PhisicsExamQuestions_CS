С самого начала я начал делать консольное приложение и реализовал в нём необходимый функционал. Сначала создал базы данных для хранения вопросов тестирования, пользователей и результатов тестирования. Далее я выделил следующие классы и добавил их в библиотеку классов.
Класс DataQuestions отвечает за выгрузку и загрузку всех вопросов в тестировании с базы данных. При написании класса я оттачивал навыки работы с файловой базой данных формата Json.
Класс FilePath отвечает за предоставление файлового пути к базам данных. Его я объявил как статический, тем самым применил навыки работы со статическими классами
Класс Question – описывает сам вопрос и его составляющие. В каждом вопросе есть описание и ответ.
 Класс ResultsTable – позволяет работать с базой данных таблицы результатов. Он отвечает за отображение таблицы всех решений разных пользователей.
 Класс Test – содержит необходимые алгоритмы для функционирования тестирования. Именно там реализован алгоритм выбора случайного вопроса из списка всех вопросов. Класс User – описывает пользователя.
Класс UserData – позволяет работать со списком пользователей и базой данных. 
С помощью этих классов я посмотрел на работоспособность всех функций приложения в консольной версии. Теперь мне надо переписать всё это на настольную версию. 


Создание Windows Forms приложения
Я начал с создания формы MainForm. Именно в этой форме пользователь отвечает на все вопросы и получает результат.
Следующим действием я реализовал MenuForm – форму, которая отвечает за главное меню приложения. В ней есть следующие режимы: вход в профиль, выход из программы, пройти тест, добавить свой вопрос, вывести таблицу результатов, удалить последний добавленный вопрос. При нажатии на кнопку с соответствующим режимом загружается соответствующая форма.
Далее я создал форму AddUserQuestionForm – форму, отвечающую за добавление пользовательского вопроса. 
Следующими по счёту были две формы, которые отвечают за вход в профиль. Если пользователь уже регистрировался, то он попадает в свой профиль, в котором отображается его статистика. Если же пользователь не был зарегистрирован, то ему надо пройти регистрацию, причём он не может зарегистрироваться с пустым логином или паролем.
После этого я создал форму ResultsTableForm. Она отвечает за добавление пользовательского вопроса. Пользователь заполняет анкету вопроса и вопрос добавляется в базу данных.
